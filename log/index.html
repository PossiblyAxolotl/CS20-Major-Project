Learning Log

Project overview

I made a game for the Arduboy and learned C++ to create it. I had to go a lot more in-depth with C++ than I had before, learning about pointers and some memory management. I learned about C++ classes and used one to create multiple instances of an enemy object in order to create a top-down dungeon crawler-ish game similar to the dungeons in the original Legend of Zelda game, and also used a third party arduboy emulator to create a web player for it.

Most proud

The thing I am the most proud of for this project is learning a programming language I've barely touched before to make it. I've used Arduino C++ before but only for more basic projects in electronics class with more of a focus on hardware over writing the code, but this time since I was making a game for existing hardware I spent more time learning about the differences and other things in C++ I didn't know as much about. I really like && and || over 'and' and 'or' from making this since it's faster to type and am no longer threatened by the existence of curly brackets and semicolons.

Most difficult

The most difficult part of this was getting used to certain parts of C++, specifically parts that are functionally different from Python like not being able to modify the length of arrays, and instead needing to delete[] an array and create a new one. I also wanted to add particles similar to these:

but since I can't modify the array and easily delete and add elements I would have to rethink the system which I didn't have time for. Other pointer related things and figuring out how to draw each sprite was also fairly difficult as I ended up using 3 different sprite drawing libraries for drawing sprites in different ways.
